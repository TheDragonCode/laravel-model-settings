# Model settings fallback and migration hardening

**Date:** 2026-07-08 02:04
**Files:** src/Services/SettingsService.php, database/migrations/2025_02_05_230000_create_model_settings_table.php, README.md, tests/Feature/HasSettings/All/BlankModelValuesTest.php, tests/Feature/Migration/ItemIdTest.php, tests/Feature/Migration/RollbackTest.php, workbench/app/Models/UuidUser.php
**Severity:** high

## Problem

`SettingsService::get()` treated blank model values as missing and fell back to defaults, but `SettingsService::all()` returned the blank model value and hid the default. The migration also could not be rolled back because it had no `down()` method. It stored `item_id` as `unsignedBigInteger`, which made the package schema too narrow for string, UUID, and ULID model keys. The JSON payload column also rejected `null`, even though the public docs describe `null` as a blank value.

## Root Cause

The fallback rule lived only in `get()`. `all()` merged raw model settings over defaults without applying the same blank-value semantics.

The migration used hand-written morph columns instead of choosing a storage type that matches the package's polymorphic API and default sentinel row. Rollback behavior was missed because the test suite only exercised fresh migration setup, not rollback.

## Solution

`SettingsService::all()` now rejects blank model values before replacing defaults. The migration now stores `item_id` as a string, makes `payload` nullable, and implements `down()` with `dropIfExists()`. The README now documents string `item_id` and nullable JSON payload.

Regression tests cover blank values in `all()`, nullable payloads, string morph item IDs, UUID model keys, and migration rollback.

## Prevention

- Keep fallback semantics consistent across single-value and bulk-read APIs.
- Test both `up()` and `down()` for package migrations.
- For polymorphic package tables, test at least one string-key model, not only auto-incrementing integer models.
- When docs mention `null` as accepted input, assert it in feature tests.
