# Робота з налаштуваннями

Один сервіс працює і зі значеннями за замовчуванням, і зі значеннями моделей. Точка входу визначає,
яка область читається або змінюється:

| Точка входу | Область |
|-------------|---------|
| `(new User)->defaultSettings()` | Значення за замовчуванням для всіх збережених моделей `User` |
| `$user->settings()` | Підсумкові налаштування одного збереженого користувача |

## Спільні значення за замовчуванням

Значення за замовчуванням застосовуються до всіх збережених моделей з однаковим morph-класом Eloquent:

```php
$defaults = (new User)->defaultSettings();

$defaults->set('timezone', 'UTC');
$defaults->set('notifications', ['email' => true]);
```

Читайте або видаляйте значення за замовчуванням через той самий сервіс:

```php
$timezone = $defaults->get('timezone');
$all = $defaults->all();

$defaults->forget('timezone');
```

Для кожного класу моделі значення за замовчуванням незалежні.

## Перевизначення для моделей

`set()` створює налаштування або замінює його поточне значення:

```php
$user->settings()->set('timezone', 'Europe/Paris');
$user->settings()->set('timezone', 'America/Toronto');
```

Змінюється лише налаштування цієї моделі. Інші моделі продовжують використовувати власне
перевизначення або спільне значення за замовчуванням.

`get()` і `all()` вибирають значення з однаковим пріоритетом:

```php
$timezone = $user->settings()->get('timezone');
$settings = $user->settings()->all();
```

`all()` повертає `Illuminate\Support\Collection` із ключами за назвами налаштувань.

`get()` приймає лише ключ. Спочатку він повертає перевизначення моделі, потім збережене значення
класу за замовчуванням, а потім `null`. Метод не приймає резервне значення від викликача.
Використовуйте колекцію, коли потрібно відрізнити відсутній підсумковий ключ від збереженого
значення:

```php
$settings = $user->settings()->all();

if ($settings->has('timezone')) {
    $timezone = $settings->get('timezone');
}
```

Наприклад, одне перевизначення замінює лише відповідне значення за замовчуванням:

```php
(new User)->defaultSettings()->set('timezone', 'UTC');
(new User)->defaultSettings()->set('locale', 'en');

$user->settings()->set('timezone', 'Europe/Paris');

assert($user->settings()->all()->sortKeys()->all() === [
    'locale' => 'en',
    'timezone' => 'Europe/Paris',
]);
```

## Видалення значення

Після видалення перевизначення моделі знову використовується значення за замовчуванням:

```php
(new User)->defaultSettings()->set('timezone', 'UTC');

$user->settings()->set('timezone', 'Europe/Paris');
$user->settings()->forget('timezone');

assert($user->settings()->get('timezone') === 'UTC');
```

Щоб видалити саме значення за замовчуванням, викличте `forget()` через `defaultSettings()`:

```php
(new User)->defaultSettings()->forget('timezone');
```

Виклик `forget()` для відсутнього ключа нічого не змінює.

## Групові зміни

Використовуйте `setMany()` і `forgetMany()`, коли в одній області потрібно змінити кілька ключів:

```php
$user->settings()->setMany([
    'timezone' => 'Europe/Paris',
    'locale' => 'fr',
]);

$user->settings()->forgetMany(['timezone', 'locale']);
```

Обидва методи приймають будь-який iterable. Перед записом `setMany()` нормалізує кожен ключ. Якщо
кілька вхідних ключів перетворюються на один збережений ключ, використовується останнє значення.
Порожні значення видаляють ключ із поточної області за тим самим правилом, що й `set()`.

Змішаний пакет `setMany()` виконує один upsert і одне видалення в транзакції. `forgetMany()` видаляє
всі перелічені ключі одним запитом. Кількість запитів залежить від типів операцій у пакеті, а не від
кількості ключів.

Використовуйте `purge()`, щоб повністю очистити поточну область:

```php
$user->settings()->purge();
```

Для `settings()` метод `purge()` видаляє лише перевизначення цього власника, після чого знову видно
збережені значення за замовчуванням. Для `defaultSettings()` він видаляє значення за замовчуванням
цього класу моделі, не зачіпаючи перевизначення моделей. Усі три групові методи повертають `void`.

## Порожні значення

`set()` і `setMany()` використовують Laravel-хелпер `blank()`. Порожнє значення видаляє налаштування,
а не зберігає його.

| Значення | Результат |
|----------|-----------|
| `null` | Видаляється |
| `''` або рядок лише з пробілів | Видаляється |
| `[]` | Видаляється |
| `0` | Зберігається |
| `false` | Зберігається |
| `'0'` | Зберігається |

Пакет не може зберегти навмисно порожнє значення через жоден із цих методів.

## Ключі налаштувань

Ключами можуть бути рядки, цілі числа або PHP-переліки, що реалізують `UnitEnum`:

```php
enum SettingKey: string
{
    case Timezone = 'timezone';
}

$user->settings()->set(SettingKey::Timezone, 'Europe/Paris');

$timezone = $user->settings()->get(SettingKey::Timezone);
```

Laravel зберігає backed enum як його базове значення, а pure unit enum — як назву варіанта. Під час
читання, заміни й видалення використовуйте той самий ключ або варіант переліку.

Пакет не перевіряє вміст ключа. Публічний API та стандартна схема дозволяють порожні ключі й ключі
лише з пробілів.

Крапки вважаються звичайними символами. Ключ `mail.from.address` — це один непрозорий ключ
налаштування, а не вкладений шлях:

```php
$user->settings()->set('mail.from.address', 'noreply@example.com');

$address = $user->settings()->get('mail.from.address');
```

## Ідентифікатори моделей

Підтримуються цілочислові, рядкові, UUID- та ULID-первинні ключі.

Для зміни налаштувань моделі потрібен збережений власник із ключем, що не дорівнює `null`. Для
незбереженої моделі `get()` повертає `null`, а `all()` — порожню колекцію без запиту перевизначень.
Її `set()`, `setMany()`, `forget()`, `forgetMany()` і `purge()` викидають
`InvalidSettingsOwnerException` до виконання запиту до сховища або читання iterable.

Ціле число `0` і рядок `'0'` зарезервовані для спільних значень за замовчуванням у версії 1.x.
Збережена модель із будь-яким із цих ключів може читати значення класу за замовчуванням, але будь-який
метод зміни викидає `InvalidSettingsOwnerException`. Інші рядкові ключі, включно з `'00'`, залишаються
допустимими.

Налаштування зберігаються для поточного morph-класу моделі. Якщо після запису налаштувань додати або
змінити псевдонім у morph map, наявні значення `item_type` доведеться оновити.
