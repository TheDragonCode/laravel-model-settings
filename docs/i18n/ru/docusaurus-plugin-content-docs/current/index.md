# Laravel Model Settings

Laravel Model Settings хранит общие настройки по умолчанию и переопределения для отдельных моделей
в отдельной таблице базы данных. Пакет подходит для случаев, когда все модели должны начинать с одного
значения, но отдельные записи могут его переопределять.

Пакет не добавляет столбцы настроек в родительские таблицы. Настройки не зависят от схемы модели и
группируются по morph-классу Eloquent-модели.

## Когда пакет подходит

| Требование | Поведение пакета |
|------------|------------------|
| Задать одинаковое начальное значение всем сохранённым моделям | Сохранить одно значение по умолчанию для класса |
| Изменить значение одной модели | Сохранить переопределение для этой модели |
| Удалить переопределение | Снова использовать значение по умолчанию для класса |
| Прочитать настройки многих моделей | Заранее загрузить одну связь со значениями по умолчанию и переопределениями |

## Порядок выбора значения

При чтении настройки пакет возвращает первое доступное значение:

1. Переопределение для сохранённой модели.
2. Значение по умолчанию для класса этой модели.
3. `null`.

| Источник | `timezone` |
|----------|------------|
| Значение по умолчанию для `User` | `UTC` |
| Переопределение для пользователя 123 | `Europe/Paris` |
| Итоговое значение для пользователя 123 | `Europe/Paris` |
| Итоговое значение для другого сохранённого пользователя | `UTC` |

Удаление переопределения снова открывает доступ к значению по умолчанию. Само значение по умолчанию
не удаляется.

## Основные операции

```php
(new User)->defaultSettings()->set('timezone', 'UTC');

$user->settings()->set('timezone', 'Europe/Paris');

$timezone = $user->settings()->get('timezone');
$settings = $user->settings()->all();

$user->settings()->forget('timezone');
```

`get()` возвращает одно итоговое значение. `all()` возвращает
`Illuminate\Support\Collection`, в которой значения по умолчанию объединены с переопределениями.

Для значений по умолчанию и переопределений доступны одинаковые четыре операции: `all()`, `get()`,
`set()` и `forget()`.

## Границы хранения

Каждая строка определяется тремя значениями:

| Значение | Смысл |
|----------|-------|
| `item_type` | Morph-класс родительской модели или псевдоним из morph map |
| `item_id` | Первичный ключ родителя или зарезервированное значение `0` для настроек класса по умолчанию |
| `key` | Имя настройки |

Благодаря этому значения по умолчанию независимы для каждого класса модели. Значение по умолчанию
для `User` никогда не становится значением по умолчанию для `Post`, даже если оба класса используют
один ключ настройки.

## Поддерживаемые модели

Пакет поддерживает Eloquent-модели с целочисленными, строковыми, UUID- и ULID-первичными ключами.
Также можно использовать Laravel morph map.

Настройки отдельных моделей относятся только к сохранённым моделям. Несохранённая модель не
наследует значения по умолчанию: `get()` возвращает `null`, а `all()` — пустую коллекцию. Вызов
`set()` или `forget()` для несохранённого владельца выбрасывает `InvalidSettingsOwnerException` до
выполнения запроса к хранилищу.

Данные настроек хранятся в JSON. Без настроенного преобразования чтение возвращает декодированные
массивы или скалярные значения. [Преобразования данных](payload-casts.md) позволяют возвращать
объекты приложения.
