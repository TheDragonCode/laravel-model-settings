# Справочник API

## Трейт HasSettings

| Метод | Возвращает | Назначение |
|-------|------------|------------|
| `settings()` | `SettingsService` | Доступ к итоговым настройкам этой модели |
| `defaultSettings()` | `SettingsService` | Доступ к общим значениям по умолчанию для класса модели |
| `modelSettings()` | Eloquent `Relation` | Загрузка значений по умолчанию и переопределений как связи |

Используйте связь `modelSettings` только с `with()`, `load()` или `loadMissing()` и как полученное
загруженное свойство. Не используйте запрос связи как альтернативный API чтения или CRUD. Читайте и
изменяйте значения через два сервиса. Во время выполнения связь представлена пакетным классом
`SettingsRelation`, основанным на связи Laravel `MorphMany`.

## SettingsService

| Метод | Возвращает | Поведение |
|-------|------------|-----------|
| `all()` | `Collection` | Возвращает значения по умолчанию, объединённые с переопределениями модели |
| `get(int\|string\|UnitEnum $key)` | `mixed` | Возвращает переопределение, значение по умолчанию или `null` |
| `set(int\|string\|UnitEnum $key, mixed $value)` | `void` | Создаёт, заменяет или удаляет пустую настройку |
| `setMany(iterable $values)` | `void` | Записывает непустые и удаляет пустые значения одним ограниченным пакетом |
| `forget(int\|string\|UnitEnum $key)` | `void` | Удаляет настройку, если она существует |
| `forgetMany(iterable $keys)` | `void` | Удаляет перечисленные ключи из текущей области |
| `purge()` | `void` | Удаляет все настройки, сохранённые в текущей области |

Методы с ключом принимают backed enum и pure unit enum. Laravel преобразует backed enum в его
базовое значение, а pure unit enum — в имя варианта.

У `SettingsService` нет резервного аргумента для `get()` и отдельного метода `has()`. Для проверки
наличия итогового ключа используйте `all()->has($key)`.

## Матрица выбора значения

| Переопределение модели | Значение класса по умолчанию | Результат `get()` | Включено в `all()` |
|------------------------|------------------------------|-------------------|--------------------|
| Есть | Есть | Переопределение | Переопределение |
| Есть | Нет | Переопределение | Переопределение |
| Нет | Есть | Значение по умолчанию | Значение по умолчанию |
| Нет | Нет | `null` | Нет записи |

Для несохранённой модели `get()` возвращает `null`, а `all()` — пустую коллекцию. Значения класса по
умолчанию наследуются только сохранёнными моделями.

## all

```php
$settings = $user->settings()->all();

$timezone = $settings->get('timezone');
$hasTimezone = $settings->has('timezone');
```

Результат — `Illuminate\Support\Collection` с ключами по ключам настроек. Для настроек модели
переопределения заменяют значения по умолчанию с тем же ключом.

## get

```php
$timezone = $user->settings()->get('timezone');
```

Результат содержит итоговое декодированное или преобразованное значение. Если переопределения нет,
используется значение по умолчанию. Если нет ни переопределения, ни значения по умолчанию,
возвращается `null`. Сигнатура намеренно не принимает второй резервный аргумент.

## set

```php
$user->settings()->set('timezone', 'Europe/Paris');
```

Метод проверяет владельца, затем выполняет update-or-create для типа модели, идентификатора модели и
ключа. Значение, которое Laravel считает пустым, удаляет строку. Проверка выполняется до выбора пути
для пустого значения. В обоих случаях загруженная связь `modelSettings` очищается, чтобы следующее
чтение не использовало устаревшие данные.

## setMany

```php
$user->settings()->setMany([
    'timezone' => 'Europe/Paris',
    'locale' => 'fr',
    'obsolete' => null,
]);
```

Ключи iterable нормализуются так же, как в `set()`. Если несколько входных ключей преобразуются в
одну строку, используется последнее значение. Непустые значения записываются одним нативным upsert,
а пустые — удаляются одним запросом. При наличии обеих групп операции выполняются в одной
транзакции. Метод проверяет владельца до чтения iterable и один раз очищает `modelSettings` после
успешного выполнения.

## forget

```php
$user->settings()->forget('timezone');
```

Для допустимого владельца метод безопасен, если ключ отсутствует. Удаление переопределения не
удаляет общее значение по умолчанию. После удаления загруженная связь очищается.

## forgetMany

```php
$user->settings()->forgetMany(['timezone', 'locale']);
```

Метод нормализует iterable, удаляет дубликаты и одним запросом удаляет только указанные ключи из
текущей области. Отсутствующие ключи ничего не меняют. Метод возвращает `void` и очищает загруженную
связь после успешного вызова, включая пустой iterable.

## purge

```php
$user->settings()->purge();
```

Для `settings()` метод удаляет все переопределения сохранённого владельца. Значения класса по
умолчанию и переопределения других владельцев не удаляются. Для `defaultSettings()` удаляются все
значения по умолчанию этого класса модели, а переопределения моделей сохраняются. Метод возвращает
`void` и очищает загруженную связь после успеха.

## defaultSettings

Сервис, возвращаемый `defaultSettings()`, предоставляет те же семь методов:

```php
$defaults = (new User)->defaultSettings();

$defaults->set('timezone', 'UTC');
$defaults->setMany(['timezone' => 'UTC', 'locale' => 'en']);
$timezone = $defaults->get('timezone');
$all = $defaults->all();
$defaults->forget('timezone');
$defaults->forgetMany(['timezone', 'locale']);
$defaults->purge();
```

## Исключения

`DragonCode\LaravelModelSettings\Exceptions\InvalidSettingsOwnerException` расширяет PHP-класс
`DomainException`. Любой изменяющий метод через `settings()` выбрасывает его до запроса к хранилищу
в двух случаях:

- Модель-владелец не сохранена, включая несохранённую модель с заранее назначенным ключом.
- Ключ сохранённого владельца равен целому числу `0` или строке `'0'`, которая конфликтует со
  служебным значением настроек класса по умолчанию в версии 1.x.

Эта проверка также выполняется до чтения группового iterable. Изменения через `defaultSettings()`
остаются допустимыми, потому что этот сервис явно выбирает область значений класса по умолчанию.
Чтение остаётся однозначным: несохранённый владелец возвращает `null` или пустую коллекцию без
запроса переопределений, а сохранённый владелец с ключом `0` может читать значения класса по
умолчанию, но не может изменять их как переопределения модели.

`DragonCode\LaravelModelSettings\Exceptions\InvalidPayloadCast` выбрасывается, если настроенное для
модели или ключа преобразование отсутствует, имеет неверный тип, не реализует поддерживаемый
контракт или не разрешается контейнером Laravel. Сообщение может содержать класс родительской модели,
ключ настройки и класс преобразования, но никогда не содержит данные настройки.

При ошибке смешанной операции `setMany()` транзакция откатывает запись и удаление. Исключение
выбрасывается дальше, а существующая загруженная связь `modelSettings` не очищается.
