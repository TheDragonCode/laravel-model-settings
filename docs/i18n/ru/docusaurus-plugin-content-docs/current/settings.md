# Работа с настройками

Один сервис работает и со значениями по умолчанию, и со значениями моделей. Точка входа определяет,
какая область читается или изменяется:

| Точка входа | Область |
|-------------|---------|
| `(new User)->defaultSettings()` | Значения по умолчанию для всех сохранённых моделей `User` |
| `$user->settings()` | Итоговые настройки одного сохранённого пользователя |

## Общие значения по умолчанию

Значения по умолчанию применяются ко всем сохранённым моделям с одним morph-классом Eloquent:

```php
$defaults = (new User)->defaultSettings();

$defaults->set('timezone', 'UTC');
$defaults->set('notifications', ['email' => true]);
```

Читайте и удаляйте значения по умолчанию через тот же сервис:

```php
$timezone = $defaults->get('timezone');
$all = $defaults->all();

$defaults->forget('timezone');
```

Для каждого класса модели значения по умолчанию независимы.

## Переопределения для моделей

`set()` создаёт настройку или заменяет её текущее значение:

```php
$user->settings()->set('timezone', 'Europe/Paris');
$user->settings()->set('timezone', 'America/Toronto');
```

Изменяется только настройка этой модели. Остальные модели продолжают использовать собственное
переопределение или общее значение по умолчанию.

`get()` и `all()` выбирают значения с одинаковым приоритетом:

```php
$timezone = $user->settings()->get('timezone');
$settings = $user->settings()->all();
```

`all()` возвращает `Illuminate\Support\Collection` с ключами по именам настроек.

Например, одно переопределение заменяет только соответствующее значение по умолчанию:

```php
(new User)->defaultSettings()->set('timezone', 'UTC');
(new User)->defaultSettings()->set('locale', 'en');

$user->settings()->set('timezone', 'Europe/Paris');

assert($user->settings()->all()->sortKeys()->all() === [
    'locale' => 'en',
    'timezone' => 'Europe/Paris',
]);
```

## Удаление значения

После удаления переопределения модели снова используется значение по умолчанию:

```php
(new User)->defaultSettings()->set('timezone', 'UTC');

$user->settings()->set('timezone', 'Europe/Paris');
$user->settings()->forget('timezone');

assert($user->settings()->get('timezone') === 'UTC');
```

Чтобы удалить само значение по умолчанию, вызовите `forget()` через `defaultSettings()`:

```php
(new User)->defaultSettings()->forget('timezone');
```

Вызов `forget()` для отсутствующего ключа ничего не меняет.

## Пустые значения

`set()` использует Laravel-хелпер `blank()`. Пустое значение удаляет настройку, а не сохраняет её.

| Значение | Результат |
|----------|-----------|
| `null` | Удаляется |
| `''` или строка только из пробелов | Удаляется |
| `[]` | Удаляется |
| `0` | Сохраняется |
| `false` | Сохраняется |
| `'0'` | Сохраняется |

Пакет не может сохранить намеренно пустое значение через `set()`.

## Ключи настроек

Ключами могут быть строки, целые числа или PHP-перечисления, реализующие `UnitEnum`:

```php
enum SettingKey: string
{
    case Timezone = 'timezone';
}

$user->settings()->set(SettingKey::Timezone, 'Europe/Paris');

$timezone = $user->settings()->get(SettingKey::Timezone);
```

Laravel сохраняет backed enum как его базовое значение, а pure unit enum — как имя варианта. При
чтении, замене и удалении используйте тот же ключ или вариант перечисления.

Пакет не проверяет содержимое ключа. Публичный API и стандартная схема допускают пустые ключи и
ключи только из пробелов.

## Идентификаторы моделей

Поддерживаются целочисленные первичные ключи, UUID и ULID. Значение `0` зарезервировано для общих
значений по умолчанию и не должно использоваться как настоящий первичный ключ модели.

Настройки сохраняются для текущего morph-класса модели. Если после записи настроек добавить или
изменить псевдоним в morph map, существующие значения `item_type` придётся обновить.
